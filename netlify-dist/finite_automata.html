Finite Automata

What is Finite Automata?

A Finite Automaton (FA) is a simple abstract machine used to recognize patterns and process input strings, forming the mathematical foundation for regular languages and many applications in computer science. It consists of a finite number of states and transitions between them, based on input symbols. If the machine ends in an accepting (final) state after reading the input, the input is accepted; otherwise, it is rejected.

Real-World Applications

  • Lexical Analysis in Compilers: Tokenizing source code into keywords, operators, and identifiers.
  • Pattern Recognition: Searching and matching patterns in text (e.g., regular expressions).
  • Digital Circuit Design: Modeling sequential circuits like traffic lights, elevators, and vending machines.
  • Text Editors & Spell Checkers: Finding/replacing patterns and checking word validity.
  • Network Protocols: Modeling connection states (e.g., TCP SYN, ACK, FIN).
  • Decision Automation: Automating processes like traffic light controllers.

Key Features of Finite Automata

  • Input: Sequence of symbols (from an alphabet, e.g., {a, b}).
  • States: Different configurations the machine can be in.
  • Transitions: Rules for moving from one state to another based on input.
  • Output: Accept or reject the input string.

Formal Definition (5-Tuple)

A finite automaton is defined as:

M = (Q, Σ, δ, q₀, F)

Where:

  • Q: Finite set of states
  • Σ: Finite set of input symbols (alphabet)
  • δ: Transition function (Q × Σ → Q for DFA, Q × Σ → 2Q for NFA)
  • q₀: Initial/start state (q₀ ∈ Q)
  • F: Set of final/accepting states (F ⊆ Q)

Types of Finite Automata

1. Deterministic Finite Automaton (DFA)
  • For each state and input symbol, there is exactly one transition.
  • No empty (epsilon) transitions.
  • Always predictable: one path for each input string.
Formal DFA Example:

Let's design a DFA that accepts all strings ending with 'a' over Σ={a,b}:

  • States: Q={q₀, q₁}
  • Alphabet: Σ={a,b}
  • Start State: q₀
  • Final State: F={q₁}
Transition Table:

How it works:
  • If the last symbol is 'a', the machine ends in q₁ (accept).
  • Otherwise, it ends in q₀ (reject).
2. Non-Deterministic Finite Automaton (NFA)
  • For a state and input symbol, multiple transitions (or none) are allowed.
  • Can have epsilon (empty string, ε) transitions.
  • Accepts the input if any path leads to a final state.
NFA Example:

Accepts strings ending with 'a' over Σ={a,b}:

DFA vs. NFA: Key Differences

FeatureDFANFA
TransitionsExactly one per symbol per stateZero, one, or many per symbol
Epsilon (ε) transitionsNot allowedAllowed
AcceptanceUnique pathMultiple possible paths
ConstructionMore difficult, more statesEasier, fewer states
EfficiencyFaster, less backtrackingMay require backtracking
PowerBoth recognize regular languagesBoth recognize regular languages

Note: Every NFA can be converted to an equivalent DFA, though the DFA may have more states.

How Finite Automata Work: Step-by-Step

  • Start at the initial state.
  • Read the input string one symbol at a time.
  • Follow transitions according to the transition function.
  • If, after reading the whole string, the automaton is in a final state, the string is accepted. Otherwise, it is rejected.

Sample Applications and Examples

  • Traffic Light Controller: States represent "Red", "Yellow", "Green"; transitions occur on timer or sensor input.
  • Elevator System: States for each floor; transitions based on button input.
  • TCP Connection: States like "LISTEN", "SYN_SENT", "ESTABLISHED", etc., transitions based on network events.
